fabric rest api 说明

服务启动

fabric rest api服务启动要求fabric网络已经正常运行. rest api服务参数支持json配置文件, 环境变量和命令行参数三种, 优先级: 配置文件 < 环境变量 < 命令行参数. 具体参数如下:

  • 配置文件

    xxxxxxxxxx
    9
    1
    {
    2
        "config": "./config.yaml",           // fabric网络连接文件
    3
        "orgname": "Org1",                   // 连接fabric网络所用的组织名
    4
        "username": "Admin",                 // 连接fabric网络所用的用户名
    5
        "orderer": "orderer.example.com",    // 连接fabric网络的orderer节点地址
    6
        "channel": "",                       // 连接fabric网络所用的通道名
    7
        "store": "./store",                  // 链码, 提案等临时文件的存储目录
    8
        "maxsize": 1048576000                // rest服务一次性最大可上传文件的大小(Byte)
    9
    }

    配置文件通过 -file 参数指定, 如无此参数将默认读取当前目录下的config.json文件

    config.yaml配置文件是rest服务连接fabric网络的必要文件, 一般包含与当前fabric组织相关的所有配置信息

  • 环境变量

    xxxxxxxxxx
    7
    1
    export FABRIC_CONN_CONFIG="/root/config.yaml"
    2
    export FABRIC_CONN_ORGNAME="Org1"
    3
    export FABRIC_CONN_USERNAME="Admin"
    4
    export FABRIC_CONN_ORDERER="orderer.example.com"
    5
    export FABRIC_CONN_CHANNEL="mychannel"
    6
    export FABRIC_CONN_STORE="/root/store"
    7
    export FABRIC_CONN_MAXSIZE=1048576000

  • 命令行参数

    xxxxxxxxxx
    1
    1
    ./fabrest -config <config> -org <orgname> -user <username> -orderer <orderer> -channel <channel> -store <store> -maxsize <maxsize>

接口说明

fabric rest api的所有rest接口都支持传递org, user和channel参数, 这三个参数默认使用服务启动时指定的相应项的配置值, 随rest请求传入新的参数将改变fabric网络连接实例. 参数传递方式遵循标准的http请求格式, 如GET请求通过在URL追加参数传递, POST请求通过FORM方式传递.

证书

CA查询

  • 描述: 查询CA服务器信息, 支持多种查询方式

  • 方法: GET

  • 路由: /ca/query/{key}

  • 参数:

    • org: 客户端连接组织名
    • user: 客户端连接用户名
    • channel: 客户端连接通道名
    • key: 查询项

  • 示例:

    xxxxxxxxxx
    4
    1
    curl -X GET 127.0.0.1:8080/ca/query
    2
    curl -X GET 127.0.0.1:8080/ca/query?key=caname
    3
    curl -X GET 127.0.0.1:8080/ca/query/all
    4
    curl -X GET 127.0.0.1:8080/ca/query/caname

用户注册

  • 描述: 向CA服务器注册新用户

  • 方法: POST

  • 路由: /ca/register

  • 参数:

    • org: 客户端连接组织名
    • user: 客户端连接用户名
    • channel: 客户端连接通道名
    • name: 注册用户名
    • type: 注册类型
    • affiliation: 注册用户组织关系
    • secret: 注册用户密码
    • maxenroll: 注册用户可登记次数
    • attribute: 注册用户自定义属性

  • 示例:

    xxxxxxxxxx
    2
    1
    curl -X POST 127.0.0.1:8080/ca/register -d "name=tim&type=client&affiliation=com.example.org1&secret=passwd&maxenroll=10&attribute=role:admin:true"
    2
    curl -X POST 127.0.0.1:8080/ca/register -d "name=tim" -d "type=client" ...

用户登记

  • 描述: 用户登记到本地, CA向用户颁发密钥和证书

    • user: 客户端连接用户名
    • channel: 客户端连接通道名
    • name: 登记用户名
    • secret: 登记用户密码

  • 示例:

    xxxxxxxxxx
    2
    1
    curl -X POST 127.0.0.1:8080/ca/enroll -d "name=tim&secret=passwd"
    2
    curl -X POST 127.0.0.1:8080/ca/enroll -d "name=tim" -d "secret=passwd"

    • user: 客户端连接用户名

    • orderer: 提交节点

  • 示例:

    xxxxxxxxxx
    1
    1
    curl -X POST 127.0.0.1:8080/channel/mychannel/create -F "file=@/root/channel-artifacts/channel.tx" -F "orderer=orderer.example.com"

  • 描述: 将指定节点加入通道

  • 方法: POST

  • 路由: /channel/{channelid}/join

  • 参数:

    • org: 客户端连接组织名
    • user: 客户端连接用户名
    • channel: 客户端连接通道名
    • peers: 加入节点列表

  • 示例:

    xxxxxxxxxx
    1
    1
    curl -X POST 127.0.0.1:8080/channel/mychannel/join -d "peers=peer0.org1.example.com,peer1.org1.example.com" //可指定加入通道的节点, 不指定则将组织内所有节点加入通道

链码安装

  • 描述: 向指定节点安装链码

  • 方法: POST

  • 路由: /chaincode/{chaincodeid}/install

  • 参数:

    • user: 客户端连接用户名
    • channel: 客户端连接通道名

  • 示例:

    xxxxxxxxxx
    1
    1
    curl -X POST 127.0.0.1:8080/chaincode/dc/install -F "file=@/root/chaincode/dc/dc.go" -F "version=1.0" -F "peers=peer0.org1.example.com"

链码实例化

  • 描述: 实例化指定节点的链码

  • 方法: POST

  • 路由: /chaincode/{chaincodeid}/instantiate

  • 参数:

    • org: 客户端连接组织名
    • user: 客户端连接用户名
    • channel: 客户端连接通道名
    • version: 链码版本
    • policy: 链码背书策略
    • peers: 实例化链码的节点列表
    • args: 链码实例化参数

  • 示例:

    xxxxxxxxxx
    1
    1
    curl -X POST 127.0.0.1:8080/chaincode/dc/instantiate -d "channel=mychannel&version=1.0&policy=and('Org1MSP.member','Org2MSP.member')&peers=peer0.org1.example.com&args=init,Admin@Org1MSP,Admin@Org2MSP"

  • 示例:

    xxxxxxxxxx
    1
    1
